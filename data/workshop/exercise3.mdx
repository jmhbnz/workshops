With a clear understanding of how Ansible Automation Platform has been deployed on to OpenShift, and now logged in to the platform Sarah gets to work automating etcd backups.

> **Background:** [etcd](https://etcd.io) is a distributed, reliable key-value store for the most critical data of a distributed system. Within Kubernetes it maintains the state of your cluster, and all resources within it. Taking regular etcd backups is a platform best practice.

![sarah-etcd-backups](/workshops/static/images/ocp-aap/sarah-etcd.png)


## 3.1 - Creating a project

To begin automating etcd backups with Ansible Automation Platform Sarah needs to create a **Project**. Projects are a foundational concept of the automation controller. Essentially they point to a git repository where your ansible playbooks are stored.

Let's create one now using the web interface.

Click on **Automation Execution** in the left menu, then click on **Projects**.

Enter the following values then click **Create**:

- **Name:** `acme-ocp-ansible`
- **Organization:** `Default`
- **Execution environment:** `Control Plane Execution Environment`
- **Source control type:** `git`
- **Source control url:** `https://github.com/jmhbnz/workshops`

<Zoom>
![cluster](/workshops/static/images/ocp-aap/aap-project.png)
 *Creating a project*
</Zoom>

After clicking the **Create** button at the bottom the project will immediately sync, you should see a `Success` status.

The source control revision refers to the commit hash that the project is currently synchronised to.

Projects can be configured to automatically sync when job templates using that project are run, or sync on a defined schedule.
