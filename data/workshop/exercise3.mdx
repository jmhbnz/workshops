With a clear understanding of how Ansible Automation Platform has been deployed on to OpenShift, and now logged in to the platform Sarah gets to work automating etcd backups.

> **Background:** [etcd](https://etcd.io) is a distributed, reliable key-value store for the most critical data of a distributed system. Within Kubernetes it maintains the state of your cluster, and all resources within it. Taking regular etcd backups is a platform best practice.

![Sarah etcd backups](/workshops/static/images/ocp-aap/sarah-etcd.png)


## 3.1 - Creating a project

To begin automating etcd backups with Ansible Automation Platform Sarah needs to create a **Project**. Projects are a foundational concept of the automation controller. Essentially they point to a git repository where your ansible playbooks are stored.

Let's create one now using the web interface.

Click on **Automation Execution** in the left menu, then click on **Projects**.

Enter the following values then click **Create**:

- **Name:** `acme-ocp-ansible`
- **Organization:** `Default`
- **Execution environment:** `Control Plane Execution Environment`
- **Source control type:** `git`
- **Source control url:** `https://github.com/jmhbnz/workshops`

<Zoom>
![Project creation](/workshops/static/images/ocp-aap/aap-project.png)
 *Creating a project*
</Zoom>

After clicking the **Create** button at the bottom the project will immediately begin a sync, after a short wait you should see a `Success` status.

The source control revision refers to the commit hash that the project is currently synchronised to.

Projects can be configured to automatically sync when job templates using that project are run, or sync on a defined schedule.

![Project sync](/workshops/static/images/ocp-aap/aap-project-sync.png)

## 3.2 - Creating a job template

By creating a project Sarah has told Ansible Automation Platform where some of her ansible content lives in git.

Sarah has previously distilled the Red Hat [etcd backup documentation](https://docs.redhat.com/en/documentation/openshift_container_platform/4.21/html/etcd/backing-up-and-restoring-etcd-data#backing-up-etcd-data_etcd-backup) into an ansible playbook. 

The playbook is available in source control here: [https://github.com/jmhbnz/workshops/blob/main/data/workshop/playbooks/etcd-backup.yaml](https://github.com/jmhbnz/workshops/blob/main/data/workshop/playbooks/etcd-backup.yaml). Take some time to review this before we proceed.

> **Note:** This playbook is intentially relatively trivial. To minimize external dependencies for the workshop all the playbook does is generate the backup on a master node. In a production situation the backup would generally be replicated outside of the cluster, often to geographically resilient storage.

To be able to run this playbook from Ansible Automation Platform we need to create a **Job Template** which will define which playbook is being run and what parameters it will be run with.

Let's create one now using the web interface.

Click on **Automation Execution** in the left menu, then click on **Templates**.

On the template list view click on the blue **Create template** button and then click **Create job template**.

Enter the following values then click **Create job template** once more:

- **Name:** `acme-ocp-etcd-backup`
- **Inventory:** `Demo Inventory`
- **Project:** `acme-ocp-ansible`
- **Playbook:** `data/workshop/playbooks/etcd-backup.yaml`
- **Execution environment:** `Control Plane Execution Environment`
- **Credentials:** `OpenShift Credential`

## 3.3 - Launching the job

Immediately after creating a job template Sarah gives it a test by launching it.

Click the blue **Launch template** button in the top right corner.

After completing, review the logs for the completed job. You should see an etcd backup being successfully created on a master node.

## 3.4 - Scheduling the job template

With the job template in place Sarah can now trigger a backup on demand, but taking a regular backup automatically is a much better practice.

For a strech exercise have a go at creating schedule for this job template now within the Ansible Automation Platform user interface.

Call the schedule `acme-ocp-etcd-backup-daily` and set a frequency of daily.


With a schedule created Sarah can tick that task off for the day! You can now move on to [Exercise 4](/workshop/exercise4), to create an end to end workflow 🚀.
