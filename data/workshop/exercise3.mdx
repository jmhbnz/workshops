## 3.1 Checkpointing the breached pod

Container checkpointing was introduced as an alpha feature in Kubernetes v1.25 and graduated to beta in Kubernetes v1.30 and is a [Generally Available feature in OpenShift 4.17 onwards](https://access.redhat.com/solutions/7008477). This functionality allows running containers to be transparently checkpointed to persistent storage and later restored to resume execution, or migrated across nodes and clusters.

The content of container checkpoints can be further analyzed with the `checkpointctl` tool. This allows to perform forensic analysis in case of security incidents (e.g., suspected compromise, data exfiltration) or application failures by inspecting the saved process memory, open files, sockets, and execution context captured in the checkpoint.

Initiate a checkpoint by copying and running the following script in your OpenShift Web Terminal:

```bash
NODE_NAME=$(oc get pods -l app=payment-gateway -o jsonpath='{.items[0].spec.nodeName}')
CURRENT_NS=$(oc project --short)

echo "==> Triggering fresh CRIU checkpoint on node: $NODE_NAME..."

API_RESPONSE=$(curl --silent --insecure \
  --header "Authorization: Bearer $(oc whoami --show-token)" \
  --request POST "${OPENSHIFT_API_URL}/api/v1/nodes/${NODE_NAME}/proxy/checkpoint/${CURRENT_NS}/payment-gateway/httpd")

TARBALL_PATH=$(echo "$API_RESPONSE" | jq -r '.items[0] // empty')

if [ -z "$TARBALL_PATH" ] || [ "$TARBALL_PATH" = "null" ]; then
  echo "[!] ERROR: Checkpoint failed. API response was:"
  echo "$API_RESPONSE"
  return 1 2>/dev/null || exit 1
fi

CHECKPOINT_FILE=${TARBALL_PATH##*/}

echo "==> Checkpoint created successfully: $CHECKPOINT_FILE"
echo "==> Streaming memory dump into Web Terminal..."

oc debug node/"$NODE_NAME" --to-namespace="$CURRENT_NS" -- \
  chroot /host /bin/sh -c "cat $TARBALL_PATH" > "/tmp/$CHECKPOINT_FILE"

echo "==> Success! Verifying saved forensic artifact in /tmp/:"
ls -lah "/tmp/$CHECKPOINT_FILE"
```

After a short wait you should see that a checkpoint has been created.

## 3.2 Exploring the checkpoint

> **Sarah:** Alright Morgan, we successfully streamed the container's memory checkpoint down to the terminal. Now we need to figure out exactly what the attacker was running.

> **Morgan:** Perfect!  We can use the `checkpointctl` tool to take a look at it.

Run the script below in your web terminal to make the `checkpointctl` binary available.

```bash
echo "==> Fetching precompiled checkpointctl toolkit..."
curl --silent --fail --location \
  'https://console-openshift-console.apps.cluster-6lrcm.dynamic2.redhatworkshops.io/api/proxy/plugin/odf-console/s3/workshop-assets-b00f6110-c782-408d-a723-77cfac0701a8/bin/checkpointctl?X-Amz-Algorithm=AWS4-HMAC-SHA256&X-Amz-Content-Sha256=UNSIGNED-PAYLOAD&X-Amz-Credential=WL3UlXvmj9ikIsaKPH3a%2F20260517%2Fnone%2Fs3%2Faws4_request&X-Amz-Date=20260517T230223Z&X-Amz-Expires=43200&X-Amz-Signature=bb11b59d011c3fe8af0fcb927f33c406dd57f0fa98778a9116934b47480cd859&X-Amz-SignedHeaders=host&x-amz-checksum-mode=ENABLED&x-id=GetObject' \
  --output /tmp/checkpointctl

chmod +x /tmp/checkpointctl
mkdir -p $HOME/bin
mv /tmp/checkpointctl $HOME/bin/checkpointctl
export PATH=$HOME/bin:$PATH

echo "==> Toolkit loaded successfully!"
checkpointctl

echo "--------------------------------------------------------"
echo "==> To inspect your captured memory dump, run:"
echo "checkpointctl inspect /tmp/\$CHECKPOINT_FILE"
```


Once `checkpointctl` is installed in your web terminal run the command below to inspect the process tree of your checkpoint(s).

```bash
checkpointctl inspect --ps-tree-env /tmp/checkpoint-payment-gateway_payment-gateway*
```

There are other parameters for `checkpointctl inspect`. Take some time to run `checkpointctl inspect --help` and explore some of the other possible flags to get a better understanding of what details we can inspect for a given checkpoint.

> **Sarah:** This is cool, I've never played with checkpoints like this before. We have some good metadata there. It shows the container runtime was CRI-O, gives us the exact timestamp of the snapshot, and lists the root process.

> **Morgan:** Check out the process tree section—there's a PID running a script called `exfil.sh`. That's our smoking gun. Let's dig into the memory pages next to extract the environment variables and network sockets they left behind.

```bash
bash-5.1 ~ $ echo "==> Extracting raw uncompressed memory images from checkpoint..."

# Create a clean working directory
mkdir -p /tmp/forensics-dump

# Extract using -xf (eXtract File) instead of -xzf
tar -xf "/tmp/$CHECKPOINT_FILE" -C /tmp/forensics-dump/

echo "==> Success! Listing raw memory image files:"
ls -lh /tmp/forensics-dump/checkpoint/
```
