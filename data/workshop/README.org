* Log in to cluster

Let's start by logging in as the cluster admin.

#+NAME: Login to openshift cluster
#+begin_src bash
source .env
oc login "${OCP_API_URL}" --username "${OCP_USERNAME}" --password "${OCP_PASSWORD}" --insecure-skip-tls-verify
#+end_src

* Cluster theming

To set up for this workshop and help build the style we're going to make some visual customisations to the OpenShift cluster to make it feel more like a customer production cluster.


** Update cluster logo

#+NAME: Update cluster logo
#+begin_src bash
oc create configmap console-custom-logo --from-file=/home/${USER}/Documents/workshops/public/static/images/fsi-security/logo.png --namespace openshift-config

cat << EOF | oc apply --filename -
apiVersion: operator.openshift.io/v1
kind: Console
metadata:
  name: cluster
spec:
  customization:
    customLogoFile:
      key: logo.png
      name: console-custom-logo
    customProductName: ACME Financial Services OpenShift Console
  perspectives:
    - id: admin
      visibility:
        state: Enabled
    - id: dev
      visibility:
        state: Disabled
EOF
#+end_src

#+RESULTS: Update cluster logo
#+begin_example
configmap/console-custom-logo created
console.operator.openshift.io/cluster configured
#+end_example


** Add an interesting notification banner

#+NAME: Create console banner
#+begin_src bash
cat << EOF | oc apply --filename -
apiVersion: console.openshift.io/v1
kind: ConsoleNotification
metadata:
  name: acme-banner
spec:
  text: ACME Financial Services Production OpenShift
  location: BannerTop
  color: 'darkblue'
  backgroundColor: '#fff'
EOF
#+end_src

#+RESULTS: Create console banner
#+begin_example
consolenotification.console.openshift.io/acme-banner created
#+end_example


* Deploy compromised workload

For the purposes of exercise 2 we need an existing project and demo workload.

#+NAME: Create demo workload
#+begin_src bash
for i in $(seq 1 30); do
  PROJECT_NAME="payment-gateway-$i"

  echo "--- Reconciling $PROJECT_NAME ---"

  # Check if project exists; if not, create it
  if ! oc get project "$PROJECT_NAME" &>/dev/null; then
    oc new-project "$PROJECT_NAME"
  else
    # If it exists, just switch to it
    oc project "$PROJECT_NAME"
  fi

  # Deploy the pod into the current project context
  cat << EOF | oc apply -f -
  apiVersion: v1
  kind: Pod
  metadata:
    name: payment-gateway
  spec:
    containers:
      - name: httpd
        image: 'image-registry.openshift-image-registry.svc:5000/openshift/httpd:latest'
EOF
done
EOF
#+end_src

* Enable collecting externalip in rhacs

We want to show how RHACS can make external traffic visible.

#+NAME: Enable rhacs externalip
#+begin_src bash
cat << EOF | oc apply --filename -
apiVersion: v1
kind: ConfigMap
metadata:
  name: collector-config
  namespace: stackrox
data:
  runtime_config.yaml: |
    networking:
      externalIps:
        enabled: ENABLED
EOF
#+end_src

#+RESULTS: Enable rhacs externalip
#+begin_example
configmap/collector-config created
#+end_example
